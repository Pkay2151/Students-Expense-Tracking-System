# Student Expense Tracking System

## Project Overview
The Student Expense Tracking System is a web-based academic project designed to help students record, organize, and monitor daily spending. The main goal is to improve financial awareness and budgeting habits through a simple, user-friendly platform.

This README summarizes and enriches the project planning work with a focus on:
- Project Integration Management
- Project Scope Management

## 1. Project Description
Effective project planning requires clear objectives, coordinated execution, and well-defined boundaries. This project applies project management principles to ensure the system is practical, achievable within a semester, and aligned with stakeholder expectations.

## 2. Project Integration Management

### Project Objectives
- Develop a simple web-based expense tracking system for students.
- Enable users to record and categorize daily expenses.
- Provide spending summaries to help users understand spending habits.

### Success Criteria
- The system records and stores user expense entries correctly.
- Core features operate reliably without major defects.
- The project is completed within the semester timeline.

### Stakeholders and Roles
| Stakeholder | Role | Interest in the Project |
|---|---|---|
| Students | Primary users | Track and manage daily expenses |
| Project Team | Developers | Successfully design and deliver the system |
| Lecturer | Supervisor and evaluator | Guide implementation and assess outcomes |

### Timeline and Milestones (4-Month Plan)
| Month / Phase | Key Activities | Milestone |
|---|---|---|
| Month 1: Planning and Analysis | Define scope, objectives, and requirements | Approved requirements and project plan |
| Month 2: System Design | Design architecture, database, and user interface | Completed system and database design |
| Month 3: Development | Implement core features (auth, expense CRUD, categories, budgets, reports) | Functional prototype |
| Month 4: Testing and Deployment | Perform testing, fix defects, finalize documentation, deploy | Final submission and demonstration |

### Constraints
- Time limitation: Project must be completed within one academic semester.
- Budget limitation: Development relies on free tools and basic hosting.
- Technical skill limitation: Team has intermediate development skills.

### Assumptions
- Team members collaborate effectively and complete assigned tasks on time.
- Required tools and frameworks remain available throughout the project.
- Stable internet access is available for collaboration, version control, and testing.

## 3. Project Scope Management

### Scope Statement
The scope defines what is included and excluded to prevent scope creep, maintain focus, and ensure on-time delivery.

### Included in Scope
- User registration and login
- Add new expense records
- Edit and delete expense records
- Expense categorization (for example: food, transport, entertainment)
- Expense summaries by category
- Budget setting (daily, weekly, monthly)
- Basic charts and tables for spending reports

### Excluded from Scope
- Bank account integration
- Real financial transactions
- Mobile application development
- Advanced financial analytics

### System Requirements
- REQ-001: The system shall allow secure user registration and login for students.
- REQ-002: The system shall enable users to record daily expenses with category, date, and description.
- REQ-003: The system shall provide predefined categories including food, transport, books, rent, utilities, and personal.
- REQ-004: The system shall support budget setting on a daily, weekly, or monthly basis.
- REQ-005: The system shall generate summaries and visual reports with charts and tables on spending patterns.
- REQ-006: The system shall ensure secure data storage using encryption and privacy measures.

### Work Breakdown Structure (WBS)
1. Project Initiation and Planning
2. Requirements Gathering and Analysis
3. System and Database Design
4. Frontend Development
5. Backend Development
6. Integration and Feature Completion
7. Testing and Quality Assurance
8. Deployment and Documentation
9. Final Review and Presentation

### Scope Change Process
Any proposed change must follow a controlled process:
1. Submit and discuss the change request within the project team.
2. Evaluate impact on time, cost, feasibility, and objectives.
3. Obtain formal approval from the project supervisor.
4. Document approved changes and update the project plan.

## 4. Risk and Mitigation Summary
| Risk | Potential Impact | Mitigation |
|---|---|---|
| Delays in development | Missed milestones | Weekly progress tracking and task reallocation |
| Limited technical expertise | Incomplete advanced features | Prioritize core functions and use proven libraries |
| Tool or internet issues | Slower collaboration and testing | Maintain backups, offline notes, and early integration checks |

## 5. Conclusion
The Student Expense Tracking System is planned using integration and scope management principles to ensure coordinated execution and realistic delivery. Integration management aligns stakeholders, goals, timeline, and resources. Scope management defines clear feature boundaries and controls changes. Together, these approaches support structured development and successful completion within the semester.

## Lessons Learned
- Clear scope boundaries are essential to avoid feature creep.
- Early stakeholder alignment reduces rework later in development.
- Milestone-based delivery improves focus and accountability.
- Simple, reliable core features are more valuable than unfinished advanced features in semester projects.
